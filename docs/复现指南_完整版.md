## 1. 项目概述

### 1.1 核心贡献

YOLO-Master 将 **Mixture-of-Experts (MoE)** 动态专家路由机制引入 YOLO 实时目标检测架构，实现了"按需计算"（compute-on-demand）范式：

- **动态路由**：根据输入复杂度自适应激活专家子集
- **异构专家**：不同感受野的卷积核专家（3×3 / 5×5 / 7×7）
- **负载均衡**：GShard 风格的辅助损失，防止专家塌陷
- **渐进式稀疏**：训练密集 → 推理稀疏，降低计算开销

### 1.2 技术架构

```
YOLO11 Backbone
    │
    ├── C3k2 Block × N
    │
    ├── ES_MOE Block (MoE层 × 3)  ← YOLO-Master 核心改进
    │       ├── DynamicRoutingLayer (Top-K 路由器)
    │       ├── Heterogeneous Experts (3×3 / 5×5 / 7×7 卷积)
    │       └── MoELoss (辅助损失, λ=0.15)
    │
    ├── SPPF Block
    │
    └── Detection Head (YOLO11 头)
```

### 1.3 可用模型

| 模型系列 | 变体 | 参数量 | 描述 |
|---------|------|--------|------|
| **YOLO-Master** | n / s / m / l / x | ~2.6M ~99M | MoE 增强的 YOLO11 |
| YOLO12 | n / s / m / l / x | ~2.2M ~57M | 最新 YOLO 架构 |
| YOLO11 | n / s / m / l / x | ~1.5M ~59M | 标准 YOLO11 |
| YOLOE | s / m / l | ~5M ~27M | YOLOE 开放词汇检测 |
| RT-DETR | n / s / m / l / x | ~18M ~211M | 实时 Transformer DETR |
| LoRA 适配版 | 各系列 | 可调 | 参数高效微调 |

## 3. 数据集准备

### 3.1 支持的数据集格式

YOLO-Master 支持两种数据集格式：**YOLO 格式**（推荐）和 **COCO 格式**。

#### YOLO 格式（推荐）

```
dataset/
├── images/
│   ├── train/        # 训练图片
│   └── val/         # 验证图片
└── labels/
    ├── train/       # 对应标签 (.txt)
    └── val/         # 对应标签 (.txt)
```

每个 `.txt` 文件对应一张图片，格式为每行一个目标：
```
<class_id> <x_center> <y_center> <width> <height> [x2 y2 x3 y3 ...]  # 多边形点
```
所有坐标归一化到 `[0, 1]` 范围。

#### COCO 格式

```json
{
  "images": [{"id": 1, "file_name": "img.jpg", "width": 640, "height": 480}],
  "annotations": [{"id": 1, "image_id": 1, "category_id": 1, "bbox": [x,y,w,h]}],
  "categories": [{"id": 1, "name": "person"}]
}
```

### 3.2 下载标准数据集

```yaml
  0: person

```

## 4. 模型训练

### 4.1 训练模式总览

YOLO-Master 支持三种训练模式：

| 模式 | 描述 | 使用场景 |
|------|------|---------|
| **全参数训练** | 从头或从预训练权重训练全部参数 | 目标任务数据集较大 |
| **MoE 训练** | 训练 MoE 模块，冻结主干网络 | 专家专业化微调 |
| **LoRA 训练** | 仅训练 LoRA 适配器 | 高效微调，资源受限 |

### 4.2 全参数训练（标准训练）

#### 使用 Python API

```python
from ultralytics import YOLO

# 加载预训练模型
model = YOLO('yolo11s.pt')

# 训练 YOLO11（标准）
results = model.train(
    data='coco8.yaml',          # 数据集配置
    epochs=100,                   # 训练轮数
    batch=16,                    # 批次大小（根据显存调整）
    imgsz=640,                   # 输入图片尺寸
    device=0,                    # GPU 设备（0 表示第一块 GPU）
    workers=8,                   # 数据加载线程数
    amp=True,                    # 启用混合精度训练（节省显存）
    cache=True,                  # 缓存图片到内存（加速但占用内存）
    patience=50,                 # 早停耐心值
    save=True,                   # 保存检查点
    save_period=10,             # 每10轮保存一次
    project='runs/detect',      # 保存目录
    name='exp',                  # 实验名称
    exist_ok=True,               # 允许覆盖同名实验
    pretrained=True,             # 使用预训练权重
    optimizer='SGD',             # 优化器（SGD/Adam/AdamW）
    lr0=0.01,                    # 初始学习率
    lrf=0.01,                    # 最终学习率 = lr0 * lrf
    momentum=0.937,              # SGD 动量
    weight_decay=0.0005,         # 权重衰减
    warmup_epochs=3,            # 学习率预热轮数
    warmup_momentum=0.8,
    warmup_bias_lr=0.1,
    close_mosaic=10,            # 最后10轮关闭 Mosaic 增强
    amp=True,                    # 混合精度
    seed=0,                      # 随机种子
    deterministic=False,         # 确定性模式（降低性能）
    multi_scale=False,          # 多尺度训练（增大 batch 波动）
    rect=False,                  # 矩形训练（减少 padding）
    cos_lr=True,                 # 余弦学习率调度
    label_smoothing=0.0,         # 标签平滑
    nbs=64,                      # 名义批次大小（用于学习率缩放）
    overwrite=True,              # 覆盖已有配置
)
```

### 4.3 YOLO-Master MoE 专项训练

YOLO-Master 是核心改进，提供了专门的 MoE 训练参数：

```python
from ultralytics import YOLO

# 加载 YOLO-Master 模型配置
model = YOLO('ultralytics/cfg/models/master/v0_1/det/yolo-master-n.yaml')

# MoE 专项训练
results = model.train(
    data='coco.yaml',           # 推荐使用完整数据集
    epochs=600,                  # MoE 通常需要更多轮次收敛
    batch=256,                   # 大批次训练
    imgsz=640,
    device=0,

    # === MoE 核心参数 ===
    moe_num_experts=8,          # 专家数量（默认 8）
    moe_top_k=2,                 # Top-K 激活专家数（默认 2）
    moe_balance_loss=0.01,      # 负载均衡损失权重（默认 0.01）
    moe_noise_std=0.1,          # 路由噪声标准差（默认 0.1）
    moe_temperature=1.0,        # 路由器温度（默认 1.0）
    moe_router_z_loss=0.001,    # Z-Loss 权重（防止路由器 logit 爆炸）
    moe_weight_threshold=0.01,  # 专家权重阈值

    # === 训练策略 ===
    freeze=5,                    # 冻结前5层（参考默认配置）
    warmup_epochs=3,
    cos_lr=True,
    lr0=0.01,
    lrf=0.01,
    amp=True,                    # 强烈建议开启混合精度
    cache=False,                 # MoE 模型较大，建议关闭缓存
)
```

## 6. 模型推理与 WebUI

### 6.1 Python API 推理

#### 单张图片推理

```python
from ultralytics import YOLO

# 加载模型
model = YOLO('yolo-master-s.pt')

# 推理单张图片
results = model(
    'path/to/image.jpg',     # 图片路径
    conf=0.25,               # 置信度阈值
    iou=0.45,               # NMS IoU 阈值
    imgsz=640,              # 输入尺寸
    device=0,               # GPU 设备
    half=False,             # 半精度
    augment=False,          # 测试时增强 TTA
    visualize=False,        # 可视化特征图
    classes=None,           # 过滤类别
    retina_masks=False,     # 高分辨率 mask
)

# 结果处理
for r in results:
    # 边界框
    boxes = r.boxes          # Boxes object
    for box in boxes:
        xyxy = box.xyxy[0].cpu().numpy()     # [x1,y1,x2,y2]
        conf = float(box.conf[0])            # 置信度
        cls = int(box.cls[0])                 # 类别ID
        print(f"BBox: {xyxy}, Conf: {conf:.3f}, Class: {cls}")

    # 分割掩码
    masks = r.masks          # Masks object
    if masks is not None:
        for mask in masks.data.cpu().numpy():
            print(f"Mask shape: {mask.shape}")  # (H, W)

    # 关键点（姿态）
    if r.keypoints is not None:
        for kp in r.keypoints.data.cpu().numpy():
            print(f"Keypoints: {kp}")

    # 保存结果图
    r.save('output.jpg')        # 保存带标注图片
    r.save('output.png')         # 保存透明背景
    r.show()                      # 显示图片
    r.plot()                      # 返回带标注的 numpy 数组
```

```python
loaded_model = load_lora_adapters(
    base_model='yolo11s.pt',
    adapter_path='lora_checkpoints/yolo11-lora-s',
)

# 合并 LoRA 权重到基模型（可导出为普通模型）
merged_model = merge_lora_weights(
    model=model,
    adapter_name='default',
)

# 导出合并后的模型
merged_model.export(format='onnx')
```

### 8.5 LoRA 各类型对比

## 10. 常见问题

### Q1: 显存不足 (CUDA OOM)

```python
# 解决策略（按优先级）：
1. 减小 batch: batch=8  →  batch=4  →  batch=2
2. 开启混合精度: amp=True
3. 关闭缓存: cache=False
4. 减小图片尺寸: imgsz=640  →  imgsz=416
5. 开启梯度检查点: gradient_checkpointing=True
6. 使用更小的模型: yolo-master-x  →  yolo-master-s  →  yolo-master-n
```

### Q2: 训练 loss 不收敛

```python
# 诊断和解决：
1. 检查数据标注：是否标注正确、类别均衡
2. 调整学习率：lr0=0.01 → 尝试 lr0=0.001
3. 检查 batch size：过小导致梯度不稳定
4. 关闭 Mosaic early：如果早期 loss 震荡，增大 close_mosaic
5. 检查图片质量：模糊/损坏的图片影响训练
6. 使用预训练权重：pretrained=True（默认为 True）
```

### Q3: mAP 低，召回率高但精确率低

- 说明模型倾向于预测过多框 → 提高 `conf` 阈值，或增加 `cls` 损失权重

### Q4: 模型过拟合

```python
# 解决方案：
1. 开启数据增强：mosaic=1.0, mixup=0.1, copy_paste=0.1
2. 增加正则化：weight_decay=0.0005 → 0.001
3. 标签平滑：label_smoothing=0.0 → 0.05
4. Dropout：dropout=0.0 → 0.1
5. 使用更大的数据集或减少训练轮次
6. 开启早停：patience=50
```
